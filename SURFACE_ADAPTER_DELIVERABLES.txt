SURFACE ADAPTER STATUS — UPDATED 2026-01-08
--------------------------------------------

Scope
- RenderSurface abstraction that decouples MTK rendering surfaces from UIKit/AppKit/SceneKit views.
- Located in MTKCore with examples and tests; no separate migration docs are shipped in this repository.

What exists today
- MTK/Sources/MTKCore/Adapters/RenderSurface.swift — public protocol + documentation.
- MTK/Sources/MTKCore/Adapters/SurfaceAdapterExamples.swift — copy-paste adapter templates (logging, dynamic wrapper, recording).
- MTK/Tests/MTKCoreTests/SurfaceAdapterTests.swift — covers adapter forwarding, content scale handling, and mock surfaces.

How it is used
- `VolumetricSceneController` exposes its active surface via the RenderSurface protocol.
- `VolumetricDisplayContainer` renders any RenderSurface in SwiftUI; adapters make it possible to swap MTKView/SCNView wrappers without UI rewrites.

Maintenance notes
- Keep new adapters in `SurfaceAdapterExamples.swift` with concise inline docs.
- Add tests alongside new adapter behaviors to keep coverage in sync with the protocol.
